About

So, we had this app that we were working on, built with AngularJS. It was a large scale app and development was coming along well. Then Angular was announced and the groundwork for a component based platform was initiated. We immediately thought, awesome, this will be a big plus for this app we were developing. It would allow us to modularize it into smaller chunks and have some separation of concerns within these web parts that make up this complex UI.

So, we reworked what we had to start using Angular. And as we started thinking in terms of components and how that altered our approach to how we architect a web application we started to see the power of components and the ability to isolate UI and logic down to just that which is needed to drive that web chunk. But, while we had the ability to isolate the code, we didn't have the ability to isolate the running of that code in the browser. We would have to spin up the app in the browser and then nav to the spot in the app to see the component in action. And this wasn't too bad in the initial development of the app as there was not much nav going on, but as the app grew it became more time consuming to get to the render of the components.

So, we created a sandbox application that would allow a dev to edit their sandbox host component and add markup and data to display a component from the main app that they were working on. Devs would have the ability to work their component in isolation and feed it the data and dependencies it needed to see it in action and to simulate the scenarios it could end up in. This worked out great for us...as a starting point. It didn't take long before the dev team wanted the ability to save their sandboxed scenarios for later use. What we needed was an app that could render stored sandboxes of components with different scenarios configured. Around this time React Storybook came out. It really embodied the same thing we wanted to have with our sandbox tool. At this same time we had already implemented a Style Guide application for our team that would handle dynamically rendering what we term "global components" from our app.

So, we set out to leverage that dynamic component loading code from our Style Guide app along with our existing sandbox app and the inspiration of what React Storybook had done to build a tool that would work with Angular to allow us to develop components in isolation. As a result, Playground for Angular was born. And from the beginning we felt that this tool would be useful to others, so we made a plan to be able to open source it once we got it all dialed in. Well, we made it to that point and are super excited to share it with the community and hope that it will help improve the component, directive and pipe building experience and productivity!
