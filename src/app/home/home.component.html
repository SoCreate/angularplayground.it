Why Use Playground for Angular?

Why Playground?

  • Work Components Without Running Your Full Application!

    Enterprise Angular applications contain lots of components with multistep workflows and many different states. This can make iterating on a component difficult and slow. Playground eliminates this problem by allowing you to instantly load a given component in any state without navigation.

  • Store 1 to n States of Your Components

    With Playground Scenarios you can stage your components with inputs and outputs leveraging different state data making it effortless to recall these states the next time you need to work on them. Once you have created multiple scenarios for a component, you can quickly navigate through each scenario to visualize the changes. This is a great way to ensure any of the changes you've made work with all other component states and scenarios.

  • Embed Your Components In External Applications

    Playground provides the ability to embed Playground Scenarios within an external style guide, visual component library, or documentation. To do this, you simply leverage Playground "Embed Mode" and iFrame your component scenario in your documentation tool.
