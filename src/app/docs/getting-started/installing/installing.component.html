Installation

For use with Angular CLI
Setting up an existing [Angular CLI](https://cli.angular.io/) application to use Playground requires one simple step: 1. run `ng add angular-playground` This will install the angular-playground package as well as set up the necessary config files for you to start developing with Playground! From here on out, running Playground is as easy as calling: ``` npm run playground ``` Navigate to `localhost:4201` and start sandboxing components!

Overview of New/Modified Files

Playground needs some new configuration files in order to work properly. Follow along to see what's been added!

A New Main Entry File

You need to pass in the appropriate selector string for your `AppComponent` (internally, Playground will set up shop in your existing `index.html`).

A New Project in the Angular CLI Config

You may want to configure the following properties: - `outputPath` A build directory (in case you want to build Playground) - `main` Points to the new `main.playground.ts` file The rest of the properties can be the same as your main project.

A Config File for Angular Playground

`sourceRoots` points at the source of your Angular application (and component libraries, if you have any). In this example, we're using the [@angular/cli](https://cli.angular.io/), so `sourceRoots` is [`./src/`]. Additionally, we need to use the name of our Angular application, as specified in our `angular.json`. If you'd like to configure @angular/cli to run on a different port, or would like to take a look at the other options available for this configuration file, see the API documentation.
