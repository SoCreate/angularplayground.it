Use Your Angular Libraries in Playground

Playground supports sandboxing your Angular Libraries in addition to your main Angular app, so you can easily work with all the sandboxed components in your Angular project. Once you have Playground set up with your main app, you will need to update 2 files: the `tsconfig.json` that is used with Playground, as well as `angular-playground.json`. Let's say our project structure looks something like this: ``` my-app ├── angular.json ├── angular-playground.json ├── package.json ├── tsconfig.json ├── ... ├── projects │ ├── my-lib │ │ ├── src │ │ │ └── ... │ │ └── ... ├── src │ ├── tsconfig.app.json │ ├── main.ts │ ├── main.playground.ts │ └── ... ``` First, you'll need to update `angular-playground.json` to include the source directory of your library.
Next, you'll need to configure a new tsconfig.json to compile everything you want to sandbox in Playground. We recommend creating a file called `tsconfig.playground.json` that lives next to `tsconfig.app.json` in your app's `/src` directory. Once this is done, update the `"playground"` entry in `angular.json` to use this new tsconfig file.
Then, in `tsconfig.playground.json`, you can start by copy-pasting the contents of `tsconfig.app.json`. Then, you'll need to include the same source directory path that you just added to `angular-playground.json`. Note that since we are now using the `"includes"` property, we need to include all the source files for the main app as well. The changes to make are demonstrated below. Finally, if your library contains unit tests, you will need to exclude those (also shown below).
**That's all!** Run Playground and you should see sandboxes from both your app and your library.
