Setup without Angular CLI

This guide assumes you already have Angular configured to work with Webpack.
Ejected from @angular/cli? Check out a [working example](https://github.com/mgmarlow/ejected-playground).
Setting up an existing application without Angular CLI requires a few steps: 1. Create an entry point for Playground: `main.playground.ts` 2. Provide configuration: `angular-playground.json` 3. Configure a build script: `package.json`

Install Angular Playground

``` npm install --save-dev angular-playground ```

Create an Entry Point

Change the entry point of your webpack configuration to point at this file, `main.playground.ts`. The `selector` property passed into `configure()` should be the selector of your root component, `app.component.ts`. If you decide to set up a separate `index.html` file in your project for Playground simply leave the selector property off of the `.configure()` call.

Add the Angular Playground Config

The above configuration file is used to configure many different Playground options (including running tests, integration with @angular/cli, etc.). See the API documentation for more information.

Add Playground to package.json

To run Playground, you will need to serve your application with your build system of choice while running the Playground CLI. We recommend using [concurrently](https://github.com/kimmobrunfeldt/concurrently). Additionally, the `--no-serve` flag is needed to prevent the Playground CLI from running an Angular CLI `ng serve` behind the scenes.
