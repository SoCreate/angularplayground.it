Run the Playground CLI

By default, the CLI will assume that its configuration file lives at the root of the project (where `package.json` is located) and is called `angular-playground.json`. If these conditions are met, you can run the CLI like so: ``` angular-playground ``` Otherwise, a configuration file must be specified: ``` angular-playground --config ./src/playground-options.json ```

Example Configuration File

The `sourceRoot` value will be a path relative to the location of your **package.json** file for your project. This will be the directory that will be looked at (it will go down recursively from there) to find your **\*.sandbox.ts** files dynamically. You will want to set up a script in your `package.json` file to handle running the cli command:

From there you can call: ``` npm run playground ``` This will compile the paths to your sandbox files in one place, allowing webpack to allocate them as dynamically imported chunks. It also configures a watch that runs the build process as you add/modify files with the `*.sandbox.ts` extension.

Cli Commands

Almost all of the configuration file options are also available as CLI commands. For more details, visit the CLI API documentation. ``` angular-playground --src ./src/ --config ./angular-playground.json ``` You can also add Playground as a build step to check your sandboxed components for errors with the `--check-errors` flag. ``` angular-playground --check-errors ```
